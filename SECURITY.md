# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1. **Do Not** disclose the vulnerability publicly
2. Send details to [security contact email]
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to:
- Confirm the vulnerability
- Assess its impact
- Plan remediation steps
- Keep you informed of our progress

## Security Measures

PPXL implements several security measures:

1. **No Credential Storage**
   - Does not store or transmit user credentials
   - No persistent data storage

2. **Network Security**
   - Uses HTTPS for all network communications
   - Respects security challenges and CAPTCHAs
   - No automated bypass of security measures

3. **Binary Security**
   - All releases are signed
   - Virus scanned before release
   - Scan results published with each release
